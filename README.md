# AI Secure Development Benchmark

## Overview

This repository is part of an independent research project exploring the security risks introduced by AI-assisted software development pipelines.

Frontier AI systems are increasingly being used to generate code, recommend dependencies, create infrastructure configurations, write tests, and support debugging. While these tools can improve productivity, they may also introduce vulnerabilities, insecure defaults, governance gaps, and hidden security debt.

This project investigates the trade-off between developer productivity and security outcomes in AI-assisted software engineering.

## Research Question

To what extent do AI-assisted software development workflows improve productivity while introducing security vulnerabilities, governance risks, or technical debt?

## Why This Matters

Software development is one of the most important real-world deployment environments for advanced AI systems. If organisations adopt AI coding assistants without appropriate safeguards, they may gain speed while increasing long-term security exposure.

This research aims to generate practical evidence that can support safer AI deployment, stronger security governance, and better decision-making around AI-assisted engineering workflows.

## Initial Research Focus

The first phase of this project will examine AI-assisted outputs across:

* Python application code
* Terraform infrastructure-as-code
* CI/CD pipeline configuration
* Dependency recommendations
* Kubernetes configuration
* Security remediation suggestions

## Planned Evaluation Criteria

Outputs will be assessed for:

* Security vulnerabilities
* Insecure defaults
* Excessive permissions
* Hardcoded secrets
* Dependency risks
* Missing validation
* Misconfigured access controls
* Maintainability and technical debt
