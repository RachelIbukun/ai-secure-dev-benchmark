# Economic Framework

## Research Question

How should organisations evaluate the trade-off between productivity gains and security risks introduced by AI-assisted software development?

## Background

Most AI adoption decisions focus on:

- Faster development
- Reduced engineering effort
- Increased output

However, productivity gains may not capture:

- Security vulnerabilities
- Remediation costs
- Incident response costs
- Governance burdens
- Technical debt

This project explores whether AI-assisted development creates hidden costs that emerge after deployment.

## Short-Term Benefits

Potential benefits include:

- Faster code generation
- Reduced development time
- Improved troubleshooting
- Increased developer throughput
- Reduced onboarding time

## Long-Term Costs

Potential costs include:

- Security vulnerabilities
- Dependency risk
- Technical debt
- Increased remediation effort
- Reduced expertise development
- Governance overhead

## Central Hypothesis

AI-assisted software development should be evaluated using both productivity metrics and security-cost metrics.

Organisations may overestimate the value of AI if they ignore downstream security and governance costs.

## Potential Policy Questions

- How should organisations measure AI productivity?
- Should AI assisted development require additional review controls?
- How should regulators think about AI generated code in critical systems?
- What level of human oversight is appropriate?
